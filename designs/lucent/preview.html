02 / Market Diagnosis

增长问题不是单点流量，而是认知、渠道与转化链路同时失衡

核心判断

这类正文页用于放一段完整的说明文字，配合 bullet points 和右侧图片。背景仍然保持白色阅读底，但边角加入光斑，让页面不再像空白文档。

  • 正文可以承载客户背景、诊断解释或方法说明。
  • bullet point 用来提炼关键动作、证据或约束。
  • 图片区域可以放素材图、项目截图、案例图或客户材料。

02 / Narrative System

先讲清业务场景，再拆解响应机制与管理动作

当一个业务机会同时牵动交付、质量和资源约束时，报告页需要先把冲突讲清楚，再把系统响应、管理判断和后续动作拆成可执行的模块。

Story setup

一个高优先级机会出现，但组织同时面对时间、质量和资源约束。

Operational conflict

理想路径被占用，关键材料延迟，改序可能带来额外风险。

Management trade-off

机会是否值得重排资源，需要被快速解释和批准。

How the Response Works

中间流程先交代响应逻辑：从信号进入系统，到形成选项、派发动作，再把执行结果沉淀为下一轮规则。

1 / Sense

Signals captured

订单、库存、交付和约束被看见。

2 / Context

Context built

把信号连接到客户、产品和资源。

3 / Decide

Options simulated

比较收益、风险和执行成本。

4 / Act

Work routed

触发责任人、系统任务和审批。

5 / Learn

Loop improves

反馈进入下一轮规则和模型。

Simple takeaway: 人判断例外，系统协调响应，组织从结果中学习。

04 / Roadmap

路线图页用阶段推进代替复杂流程图

Phase 01 / Weeks 1-2

统一诊断口径

整理客户目标、业务约束、核心假设和现有证据，形成问题地图。

Phase 02 / Weeks 3-5

重建价值叙事

拆解角色、场景和购买理由，定义统一但可分发的内容模块。

Phase 03 / Weeks 6-9

渠道试点验证

选择两个重点渠道，测试新叙事、新证据和转化动作的组合效果。

Phase 04 / Weeks 10-12

复盘与扩展

把验证结果转成组织节奏、指标看板和下一轮资源配置建议。

04 / Recommendation

建议页把结论、理由和下一步动作放在同一张页面上

Recommendation

先做 90 天叙事与证据重建试点

以一个重点客群和两个重点渠道作为验证范围，避免一次性重构所有市场动作。

90

days to validate first operating loop

Rationale

投入前先降低策略不确定性

当前最大风险不是单个渠道选择，而是购买理由和证据资产不匹配。

  • 减少无效投放。
  • 提升销售解释效率。
  • 建立可复用内容模块。

Next Steps

下一步动作

  1. 1

    确认试点客群和成功指标。

  2. 2

    完成价值主张与证据资产梳理。

  3. 3

    启动渠道测试并按双周复盘。
